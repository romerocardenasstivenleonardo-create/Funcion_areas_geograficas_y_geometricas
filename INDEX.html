🗺️ Áreas Geográficas y Geométricas en pgAdmin

📐 En PostgreSQL + PostGIS (extensión geoespacial) y en el motor nativo de tipos geométricos, existen funciones poderosas para calcular áreas. La diferencia clave:

✅ Geométricas (planas): trabajan sobre coordenadas Cartesianas (proyección plana). El área se calcula en unidades cuadradas del sistema de referencia (metros, pies, grados²…).

🌍 Geográficas (elipsoidales): consideran la curvatura terrestre. Usan geography de PostGIS, devuelven área en metros cuadrados sobre el elipsoide (WGS84).

📌 A continuación se detallan las funciones más útiles con su nombre, descripción y ejemplos claros para ejecutar en pgAdmin (Query Tool).

ST_Area(geometry) PostGIS · área planimétrica

📐 Descripción: Calcula el área de una geometría poligonal (POLYGON, MULTIPOLYGON) en el sistema de coordenadas asociado. Si la geometría está en un CRS proyectado (como UTM, Web Mercator), el área se devuelve en las unidades lineales al cuadrado (metros², pies², etc.). Si usa coordenadas geográficas (longitud/latitud, SRID 4326), el resultado será en grados², lo cual carece de sentido físico real → se recomienda convertir a geography o proyectar.

✔️ Requisito: Extensión PostGIS habilitada (CREATE EXTENSION postgis;).

💡 EJEMPLO MUY CLARO Área de un cuadrado en metros (SRID 32630 - UTM)
-- 1. Crear un cuadrado de 100x100 metros (coordenadas UTM, zona 30N)
WITH mi_parcela AS (
    SELECT ST_GeomFromText('POLYGON((500000 4640000, 500100 4640000, 500100 4640100, 500000 4640100, 500000 4640000))', 32630) AS geom
)
SELECT 
    ST_Area(geom) AS area_metros_cuadrados,
    ST_Area(geom) / 10000 AS area_hectareas
FROM mi_parcela;
✅ Resultado (pgAdmin):
area_metros_cuadrados = 10000.0 | area_hectareas = 1.00
👉 Interpretación: el polígono de 100m x 100m genera exactamente 10.000 m².
🌍 Ejemplo con coordenadas geográficas (evitar grados²)
-- Geometría en WGS84 (longitud/latitud) → ST_Area da grados², sin significado real.
SELECT ST_Area(ST_GeomFromText('POLYGON((-71.5 -33.0, -71.4 -33.0, -71.4 -32.9, -71.5 -32.9, -71.5 -33.0))', 4326)) AS area_grados_cuadrados;
-- Resultado: ~0.008 grados², lo cual no es útil. Para área real use ST_Area(geography).
ST_Area(geography) PostGIS · área geográfica real

🌐 Descripción: Versión sobre tipo geography. Calcula el área de una superficie poligonal sobre el esferoide terrestre (por defecto WGS84). Devuelve el área en metros cuadrados, teniendo en cuenta la curvatura de la Tierra. Ideal para datos con coordenadas geográficas (lat/lon) sin necesidad de reproyectar.

✔️ Acepta polígonos, multipolígonos. Parámetro opcional use_spheroid (true por defecto). Si se usa false calcula sobre esfera perfecta, más rápido pero ligeramente menos preciso.

⚠️ Nota: Las geometrías deben ser válidas y no cruzar el antimeridiano (puede usarse ST_ShiftLongitude).

🌎 EJEMPLO CLARO Superficie de un cuadrado de 1° x 1° cerca del ecuador
-- Polígono de 1 grado de lado en el ecuador (longitud -75° a -74°, latitud 0° a 1°)
SELECT ST_Area(
    ST_GeogFromText('POLYGON((-75 0, -74 0, -74 1, -75 1, -75 0))')
) AS area_metros_cuadrados;
📏 Resultado aproximado: 12.312.000 m² (≈ 12.312 km²).
✨ Nota: debido a la curvatura, varía ligeramente según el esferoide. En un plano sería 111.319 m de lado → área ~12.392 km²; la diferencia muestra por qué es clave usar geography.
🏞️ Comparación con el área plana (grados²) vs geográfica
-- Mismo polígono en WGS84 (geometría) vs geografía
WITH region AS (
   SELECT ST_GeomFromText('POLYGON((-71.2 -34.5, -70.8 -34.5, -70.8 -34.1, -71.2 -34.1, -71.2 -34.5))', 4326) AS geom
)
SELECT 
    ST_Area(geom) AS area_grados_cuadrados,
    ST_Area(geom::geography) AS area_real_m2,
    ROUND(ST_Area(geom::geography) / 1000000, 2) AS area_km2
FROM region;
✅ Salida típica: area_grados_cuadrados ~0.24 | area_real_m2 ~ 1.881.500.000 m² | area_km2 ~ 1.881.50 km²
🗺️ La conversión a geography evita el sinsentido de los grados².
area(polygon) PostgreSQL nativo · geometría plana

⬛ Descripción: Función nativa de PostgreSQL para el tipo de dato polygon (no requiere PostGIS). Calcula el área de un polígono en el plano cartesiano. Las coordenadas se interpretan como números reales (x,y) y el área se devuelve en unidades cuadradas acordes a las coordenadas suministradas. Es útil para geometrías 2D simples sin necesidad de extensiones espaciales.

🔹 Se basa en la fórmula del área del polígono (fórmula del shoelace). El resultado siempre es positivo.

Nota: Los polígonos se definen con sintaxis '((x1,y1),(x2,y2),...)' y el último punto debe cerrar implícitamente el anillo.

📐 EJEMPLO MUY CLARO Área de un triángulo y un cuadrado
-- Triángulo rectángulo de base 4 y altura 3
SELECT area(polygon '((0,0),(4,0),(0,3))') AS area_triangulo;

-- Cuadrado de lado 5
SELECT area(polygon '((0,0),(5,0),(5,5),(0,5))') AS area_cuadrado;
📌 Resultados:
area_triangulo = 6.0   (0.5 * 4 * 3 = 6)
area_cuadrado = 25.0  (5 * 5 = 25)
🏠 Polígono irregular en el plano (coordenadas arbitrarias)
-- Polígono con forma de L (rectángulos unidos)
SELECT area(polygon '((0,0),(4,0),(4,2),(2,2),(2,4),(0,4))') AS area_forma_L;
🧮 Cálculo: área = 4*2 + 2*2 = 8+4 = 12 unidades².
Resultado SQL: 12.0
area(circle) PostgreSQL nativo · círculos planos

⚪ Descripción: PostgreSQL también ofrece el tipo circle (centro y radio). La función area(circle) devuelve el área plana de un círculo: π * radio². Resultado en las mismas unidades que el radio al cuadrado. Es una función geométrica básica pero muy útil para geometrías euclidianas.

Formato de entrada: '<(x,y>, radio)' o '((x,y), radio)'. El radio es numérico.

🔘 EJEMPLO MUY CLARO Círculo de radio 3 unidades
-- Crear un círculo con centro (0,0) y radio 3
SELECT area(circle '((0,0),3)') AS area_circulo;
📐 Resultado exacto (en unidades de radio²): 28.27433388230814
(π × 3² = π × 9 ≈ 28.274)
🏙️ Comparación de áreas: círculo vs cuadrado circunscrito
WITH figuras AS (
    SELECT 
        area(circle '((10,10),5)') AS circulo_area,
        area(polygon '((5,5),(15,5),(15,15),(5,15))') AS cuadrado_area
)
SELECT 
    circulo_area,
    cuadrado_area,
    ROUND((circulo_area / cuadrado_area) * 100, 2) AS porcentaje_circulo_cuadrado
FROM figuras;
📊 Salida: circulo_area = 78.5398, cuadrado_area = 100, porcentaje ≈ 78.54%
✔️ Un círculo inscrito en un cuadrado ocupa ~78.5% de su área.
ST_Area + ST_Transform Conversión entre geometrías proyectadas

🔄 Descripción: Cuando se tienen geometrías geográficas almacenadas como geometry (SRID 4326), podemos calcular el área real usando ST_Area(ST_Transform(geom, destino_proyectado)) o convertir a geography. Esta combinación es eficiente si se requiere trabajar en un CRS local (metros).

Recomendación: Para máxima precisión, elegir un sistema de coordenadas proyectado adecuado para la región de interés (por ejemplo UTM).

🗺️ Área real a partir de geometría WGS84 usando ST_Transform
-- Ejemplo: polígono en Santiago, Chile (WGS84). Transformamos a UTM 19S (SRID 32719)
SELECT ST_Area(
    ST_Transform(
        ST_GeomFromText('POLYGON((-70.65 -33.45, -70.60 -33.45, -70.60 -33.40, -70.65 -33.40, -70.65 -33.45))', 4326),
        32719
    )
) AS area_metros_cuadrados;
🧭 Resultado orientativo: ~ 22.750 m² (depende del polígono exacto).
★ Este valor coincide con el área real en el terreno, ya que UTM usa metros.
⚡ Alternativa directa con geography (más simple)
SELECT ST_Area( ST_GeogFromText('POLYGON((-70.65 -33.45, -70.60 -33.45, -70.60 -33.40, -70.65 -33.40, -70.65 -33.45))') );
📍 Da el mismo resultado (metros²) sin reproyección explícita, usando el elipsoide.
🧠 Resumen práctico en pgAdmin: Usa ST_Area(geometry) para planos proyectados, ST_Area(geography) para áreas terrestres reales con lat/lon, y las funciones area(polygon)/area(circle) para geometrías nativas de PostgreSQL sin PostGIS.
🔍 Todos los ejemplos pueden ejecutarse directamente en la ventana de consultas de pgAdmin. Asegúrate de tener PostGIS instalado si utilizas funciones ST_.
