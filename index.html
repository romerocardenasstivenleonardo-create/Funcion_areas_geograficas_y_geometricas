La función de áreas geográficas y geométricas en bases de datos

Cálculo, almacenamiento y análisis espacial

Stiven Leonardo Romero Cárdenas

Introducción

  • Las bases de datos modernas almacenan datos espaciales (ubicaciones, formas, regiones).
  • Dos conceptos clave: áreas geométricas (planas) y áreas geográficas (sobre la Tierra curva).
  • Objetivos:
    • Entender cómo se representan.
    • Conocer las funciones para calcular y operar con áreas.
    • Ver casos de uso reales.

Tipos de datos espaciales

Geometría (Geometry)

  • Plano cartesiano (x, y)
  • Unidades planas (metros, grados planos)
  • Ideal para mapas locales, planos, UTM

Geografía (Geography)

  • Curvatura terrestre (esferoide)
  • Superficie real, lat/lon (WGS 84)
  • Análisis globales, rutas, parcelas grandes

¿Cómo se almacenan las áreas?

  • Una región se guarda como polígono o multipolígono.
  • El área no se almacena como columna fija, se calcula dinámicamente.
CREATE TABLE parcelas (
  id SERIAL PRIMARY KEY,
  nombre TEXT,
  geom GEOMETRY(POLYGON, 32618) -- UTM zona 18N
);

Funciones principales de área

  • ST_Area(geometría) → área en unidades del sistema de coordenadas (ej. m² si es UTM).
  • ST_Area(geografía) → área en m² considerando curvatura.
SELECT nombre, ST_Area(geom) AS area_m2 FROM parcelas;
SELECT nombre, ST_Area(geog) AS area_m2 FROM paises;

También existen ST_Length, ST_Perimeter para bordes.

Ejemplo práctico con geometría

Escenario: Parcelas agrícolas en Colombia (Magna-Sirgas / UTM).

INSERT INTO parcelas (nombre, geom)
VALUES ('Finca El Edén',
  ST_GeomFromText('POLYGON((...))', 32618));

SELECT nombre, ST_Area(geom) / 10000 AS ha
FROM parcelas;

Resultado exacto dentro de la proyección plana.

Ejemplo práctico con geografía

Escenario: Áreas de departamentos en Colombia (SRID 4326).

SELECT depto,
       ST_Area(geog::geography) / 1e6 AS km2
FROM limites_departamentales;

Si usáramos ST_Area(geom) en 4326, obtendríamos "grados cuadrados", sin sentido físico.

¿Qué función usar? Criterios

Usar GEOMETRYUsar GEOGRAPHY
Datos locales o regionalesDatos globales o continentales
Proyección plana adecuada (UTM, Lambert)Coordenadas geográficas WGS84
Cálculos rápidos, operaciones euclidianasPrecisión en distancias y áreas reales
Catastro, planos de construcciónRutas marítimas, áreas entre países

Índices espaciales y rendimiento

  • Para grandes conjuntos: índices GiST.
CREATE INDEX idx_geom ON parcelas USING GIST (geom);
  • Permiten buscar rápido "parcelas con área > 1 ha" usando el bounding box.
  • Las funciones de área se aplican después de filtrar.

Casos de uso reales

  • 🏠 Catastro: superficies de predios para impuestos.
  • 🌳 Medio ambiente: monitoreo de deforestación.
  • 🚜 Agricultura de precisión: medir lotes, estimar insumos.
  • 🚚 Logística: áreas de cobertura de centros de distribución.
  • 🏥 Salud pública: extensión de focos epidémicos.

Buenas prácticas y recomendaciones

  • ✔️ Verifica el SRID de tus datos.
  • ✔️ Usa proyecciones con unidades en metros para áreas con geometry.
  • ✔️ Prefiere geography para escalas continentales/globales.
  • ✔️ ST_Area(geog, use_spheroid:=true) para máxima precisión; false para aproximación rápida.
  • ✔️ Evita cálculos de área repetidos; materializa si es necesario.

Conclusión

  • Las bases de datos espaciales gestionan y analizan áreas con funciones robustas.
  • La diferencia entre geometría y geografía es crucial para resultados correctos.
  • ST_Area es la puerta a innumerables aplicaciones territoriales.
  • Con índices y buenas prácticas, los análisis son rápidos y escalables.
